# Docs Directory

This directory contains supporting documentation for the development process of the AI Slot Machine SWE Warm-Up.

## Purpose

The goal of this folder is to capture important technical and process-related information that supports the project but does not belong directly in the source code or planning artifacts.

## Files

- `architecture.md` — documents the structure of the application, including major components, responsibilities, and how different parts of the system interact
- `prompting-notes.md` — records useful prompting patterns, observations, and lessons learned while working with the coding assistant
- `manual-edits.md` — documents any manual code changes made after first attempting to solve the issue through AI prompting

## Why this matters

This assignment is not only about building a better slot machine, but also about understanding how AI fits into a software engineering workflow. These documents help us track:

- how the system was designed
- how prompting evolved during development
- when and why human intervention was necessary

## Expectations

Documentation in this directory should be:

- clear and readable
- updated as the project evolves
- specific enough to explain decisions and tradeoffs
- useful to teammates, TAs, and future reviewers of the repo

## Notes

This folder complements the planning artifacts in `plan/` and the final conclusions in `final-report/`. Together, these documents provide a fuller picture of both the product and the engineering process behind it.
