# Candidates Directory

This directory contains various slot machine implementation candidates developed during the AI Slot Machine project.

## Purpose

The candidates represent different approaches, iterations, and experiments in building the slot machine using AI assistance. They show the evolution of the project from initial prototypes to refined implementations.

These candidates are essentially entries — individual attempts at creating a functional slot machine, each with unique features, styles, and implementations.

## Organization

- `candidate-01/` to `candidate-16/` — Individual candidate implementations, each with their own HTML, scripts, styles, and stats
- `akatsuki-candidates/` — Subset of candidates themed around "Akatsuki" (candidates 13-16)
- `spongebob-candidates/` — Subset of candidates themed around "Spongebob" (candidates 05-06)
- `Final-Candidates/` — Selected final candidates organized by theme (Akatsuki and Spongebob)

## Candidate Structure

Each candidate typically includes:

- `index.html` — Main application entry point
- `scripts/` — JavaScript files for game logic
- `styles/` — CSS for styling and layout
- `stats.md` — Performance and feature statistics
- Additional files like `package.json`, `esbuild.config.js`, or `assets/` depending on the candidate

## Winner Selection

The final implementation in `src/` is based on `candidate-06`, selected as the winning candidate after evaluation.

## How to Explore

- Open any `index.html` in a browser to run that candidate
- Review `stats.md` for each candidate's metrics and features
- Compare implementations across candidates to see different approaches
